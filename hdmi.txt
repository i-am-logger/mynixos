● Here's what I found:

  Elgato 4K X Audio Specs

  - Native sample rate: 48kHz (44.1kHz may cause stutter/drift)
  - Format: Stereo only (no surround sound)
  - HDMI Audio: PCM stereo

  So 48kHz is correct - that's not the issue.

  The Real Problem

  Based on research, this is a known AMD kernel driver bug where:

  1. The eld_valid flag is never set to 1 for capture cards like Elgato
  2. Since kernel 4.15+, Linux checks this flag before enabling HDMI audio
  3. The GPU sends video fine, but never starts the audio stream because it thinks nothing is connected

  Key findings:
  - https://www.phoronix.com/news/AMD-HDMI-Audio-Linux-6.1-Fix - fixes were developed
  - https://bugs.launchpad.net/ubuntu/+source/linux/+bug/1834771 - "root cause is the graphic driver amdgpu"
  - https://forums.linuxmint.com/viewtopic.php?t=358925 - Same issue: "No Audio HDMI Passthrough Elgato"
  - https://bbs.archlinux.org/viewtopic.php?id=282239 - HDMI Audio broken after kernel 6.1.1

  Potential Fixes

  1. Physical replug - Sometimes unplugging/replugging HDMI forces ELD re-read
  2. EDID override - Force a custom EDID with audio descriptors
  3. Kernel downgrade/upgrade - Some versions have fixes, others broke it again
  4. Wait for upstream fix - Granite Ridge is very new, driver support is still maturing
